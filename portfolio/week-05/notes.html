Errors, Exceptions, and Warnings

When it comes to programming, you are bound to come across errors. This is mainly caused by the possible reasons:

  • System error ― there is a problem with the system or external devices with which the program is interacting.
  • Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
  • User error ― the user has entered data incorrectly, which the program is unable to handle.

Writing clean code should always be the main focus of a programmer. As you write code, you should be fully aware of the possiblity that the code could give a potential error; this in turn could affect the user experience.

Stack Traces

A stack trace is a sequence of functions or method calls that lead to the point where the error occurred. Stack trace works backwards from the point the error started to identify the sequence.


Warnings

A warning occurs when there's an error in the code, but it is not enough to cause the program to crash. Even if the program is running incorrectly and not crashing, it could cause some issues at some point.

Object-Oriented Programming (OOP) in JavaScript

Programming that involves separating the code into objects that have properties and methods. This helps to keep related pieces of code encapsulated in objects "that maintain state throughout the life of the program."

Object-Oriented Programming

There are three main concepts in OOP: inheritance, polymorphism and encapsulation.

Encapsulation

The process of binding the data (variables) with the functions acting on that data. This will make it possible to control the data and validate it.

Polymorphism

The concept of reusing a single piece of code multiple times. By using polymorphism, one type of object can have different behavior.

Inheritance

The method of creating new classes on the basis of already existing classes. This allows you to create new functionality whilst taking advantage of the functionality of an already existing class.
